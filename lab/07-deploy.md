# ラボ7: デプロイ — おまかせも、決定論的にも
**テーマ**: Copilot CLI でのデプロイ体験

## 前提条件

- [ラボ環境セットアップガイド](./setup/README.md)を完了していること
- ラボ2で生成した以下が存在すること:
  - `infra/` にBicepファイル
  - `azure.yaml`（azd プロジェクト構成）
  - `hooks/`（preprovision / postprovision スクリプト）
- ラボ6で生成した検証用アプリが `src/verification-app/` に存在すること

## シナリオ

ラボ1〜6で作成した Bicep テンプレートと検証用アプリを、azd で Azure にデプロイする。デプロイ、動作確認までを Copilot CLI で行う。

---

## ラボ手順

### ステップ1: 事前設定、チェックとデプロイ

**Copilot CLIに入力:**
```
azdでインフラとアプリをデプロイして。PoC向けで
```

**期待する結果:**
- スキルやMCPサーバを活用してデプロイが行われる
- `azd up` コマンド（または `azd provision` + `azd deploy`）でインフラとアプリがデプロイされる
  - インフラは `azd provision`
  - アプリは `azd deploy`
  - `azd up` は、それを合わせて実行

> [!WARNING]
> 初回デプロイは時間がかかる。

---

### ステップ2: デプロイ結果の確認と検証用アプリの動作確認

**Copilot CLIに入力:**
```
作成されたリソースの一覧と状態を確認して
```

**期待する結果:**
- Azure MCP Server や Azure CLI などでリソースが確認される
- 作成されたリソースの一覧と状態が表示される


**検証用アプリの動作確認:**

検証用アプリのエンドポイントを、Copilot CLI に確認させる。

**Copilot CLIに入力:**
```
デプロイしたアプリの /health と /check-db エンドポイントを叩いて、結果を見せて
```

**期待する結果:**
- App Service の URL が特定され、各エンドポイントに HTTP リクエストが送信される
- レスポンスの JSON が整形、もしくは要約されて表示される

---

## まとめ

- デプロイ前に IaC とアプリの整合性を検証することで、デプロイ後の試行錯誤を減らせる
- `azd up` でインフラと検証用アプリを一括デプロイ → 動作確認まで実施
- 問題が発生したら Copilot CLI に診断と修正を依頼する
