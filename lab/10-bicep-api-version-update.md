# ラボ10: (別軸) Bicep API バージョンの最新化
**テーマ**: 日々の運用メンテナンスを AI で効率化する

## 前提条件

- [ラボ環境セットアップガイド](./setup/README.md)を完了していること
- ラボ2（IaC生成）が完了し、`infra/` にBicepファイルが存在すること

## シナリオ

Bicep で定義した Azure リソースには API バージョンが指定されている。API バージョンは Azure の進化とともに更新され、古いバージョンを使い続けると以下の問題が生じる:

- 新しいプロパティや機能が利用できない
- 非推奨バージョンの廃止に伴うデプロイエラー

しかし、API バージョンの更新は地味で手間がかかる作業である。リソースごとに最新の GA 版を調べ、破壊的変更を確認し、Linter を通す — これを Copilot CLI とスキルで自動化する。

---

## ラボ手順

### ステップ1: スキルの確認

`bicep-api-version-updater` スキルが有効であることを確認する。

**Copilot CLIに入力:**
```
/skills list
```

**期待する結果:**
- プロジェクトスキル一覧に `bicep-api-version-updater` が表示される

> [!TIP]
> このスキルは `.github/skills/bicep-api-version-updater/SKILL.md` で定義されている。スキルには更新フロー（プレビュー版のスキップ判定、Breaking Changes 確認、Linter 検証）が詳細に記述されており、Copilot CLI はこの手順に従って作業を進める。

---

### ステップ2: API バージョンの最新化

Bicep ファイルの API バージョンを最新化する。

**Copilot CLIに入力:**
```
@infra/ 配下の Bicep で使われている API バージョンを更新して
```

**期待する結果:**
- `bicep-api-version-updater` スキルが自動的に呼ばれる
- 各リソースの API バージョンが `az provider show` で最新 GA 版と比較される
- 更新が必要なリソースは API バージョンが書き換えられる
- `az bicep build` で Linter 検証が行われ、BCP081 警告が出た場合は元のバージョンに戻される
- プレビュー版を使用しているリソースがあれば、GA 移行可否の分析が行われる

> [!NOTE]
> ラボ2 で生成した Bicep の API バージョンがすでに最新の場合、全リソースが「スキップ（既に最新）」となる。その場合でも、スキルの動作フロー（バージョン取得 → 比較 → 判定）を確認できる。

---

### ステップ3: 結果の確認

**期待する結果:**
- 更新サマリーテーブルが出力される（全リソースの更新状況を一覧表示）
- 更新されたリソースは「✅ 更新」、既に最新のリソースは「⏭️ スキップ（既に最新）」と表示される
- BCP081 警告でスキップされたリソースがあれば「⚠️ スキップ（BCP081警告）」と理由が表示される
- プレビュー版リソースがあれば、GA 版への移行可否が分析される

> [!IMPORTANT]
> - サマリーテーブルで全リソースの状態を確認する
> - 更新されたリソースがある場合、`git diff` で変更内容を確認する
> - BCP081 警告でスキップされたリソースは、Bicep CLI の更新後に再実行すれば更新可能になる場合がある

---

## まとめ

- Bicep の API バージョン最新化という定型的な運用作業を、スキルで手順化し Copilot CLI に委任できる
- スキルにより「最新 GA 版の取得 → 破壊的変更の確認 → 仮適用 → Linter 検証 → 確定/ロールバック」の一連のフローが自動化される
- プレビュー版の扱い（スキップ + GA 移行可否分析）や BCP081 警告時のロールバックなど、判断が必要な場面もスキルがガイドする
- 運用ナレッジをスキルとしてコード化することで、チーム全体で一貫した作業品質を維持できる
