# ラボ11: (別軸) Excel パラメーター管理表 → IaC（Bicep）生成
**テーマ**: Excelでのリソースパラメーター管理からIaCへの移行をAIで加速

## 前提条件

- [ラボ環境セットアップガイド](./setup/README.md)を完了していること
- サンプルExcelファイルが `lab/setup/azure-resource-params.xlsx` に存在すること
  - 存在しない場合: `uv run lab/setup/create_sample_excel.py` で生成

## シナリオ

リソースのパラメーター（VM名、サイズ、ネットワーク構成など）を**Excelで管理**し、**Azureポータルで手入力**してリソースを作成・維持しているユーザーがいる。この運用は以下の問題を抱えている:

- Excelとポータルの設定のズレ（ドリフト）
- 手作業によるオペレーションミス
- 変更履歴の追跡が困難
- 新機能に追従できない
- 環境の再現性がない

このラボでは、既存の Excel パラメーター管理表を Copilot CLI で読み込み、Bicep コードに変換する。

## サンプルExcelの構成

`lab/setup/azure-resource-params.xlsx` には以下のシートが含まれる:

| シート | 内容 |
|--------|------|
| VNet | 仮想ネットワーク定義（名前、アドレス空間、リージョン） |
| Subnet | サブネット定義（名前、所属VNet、アドレスプレフィックス） |
| NSG | ネットワークセキュリティグループ定義 |
| NSGルール | NSGのセキュリティルール詳細 |
| VM | 仮想マシン定義（名前、サイズ、イメージ、ディスク、ネットワーク） |

## ラボ手順

### ステップ1: Excel 読み込みと課題分析

**Copilot CLIに入力:**
```
@azure-resource-params.xlsx を読み込んで、各シートの内容を整理して見せて。どんなAzureリソースがどういう構成で管理されているか教えて
```

**期待する結果:**
- openpyxl や pandas で Excel ファイルが読み込まれる
- 各シート（VNet、Subnet、NSG、NSGルール、VM）の内容が表形式で表示される
- リソース間の関係性（VNet → Subnet → NSG → VM）が整理されて説明される

続けて、課題の分析と IaC 移行の提案を依頼する:

**Copilot CLIに入力:**
```
このExcel管理の運用にはどんなリスクがある？IaC（Infrastructure as Code）に移行するメリットを、この具体的なリソース構成を例に説明して
```

**期待する結果:**
- Excel 管理の課題（ドリフト、手作業ミス、変更履歴の欠如、再現性の問題）が指摘される
- IaC 化のメリットがこのリソース構成に即して説明される
- 移行の方向性が提案される

---

### ステップ2: Bicep 生成とレビュー

**Copilot CLIに入力:**
```
このExcelのパラメーター管理表の内容をBicepテンプレートに変換して。以下の方針で:
- Excelの値をそのままパラメーターとして使う
- 一般的な推奨、ガイドラインに従う
- tmp/ に出力して
```

**期待する結果:**
- Azure MCP Server の `get_azure_bestpractices` ツールで推奨事項が確認される
- Azure MCP Server の `bicepschema` ツールで Bicep スキーマが参照される
- Excel の各シートのデータが読み取られ、Bicep リソース定義に変換される
- `main.bicep` ファイルが生成される
- VNet → Subnet → NSG → VM の依存関係が正しく表現される

続けて、生成された Bicep をレビューする:

**Copilot CLIに入力:**
```
生成されたmain.bicepをレビューして。Excelの元データと照合して漏れや差異がないか確認し、セキュリティの観点でもアドバイスをちょうだい
```

**期待する結果:**
- 生成された Bicep ファイルと Excel の元データが比較される
- リソース定義の漏れや値の不一致がないか確認される
- NSG ルールのセキュリティ面のアドバイスが提示される（例: 送信元の絞り込み提案など）
- 必要に応じて修正が提案される

---

- Azure MCP Server で SKU やプロパティを検証
- 元データとの照合で差異を確認
