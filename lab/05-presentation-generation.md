# ラボ5: Excel/PowerPointの読み書き — azure-pptx Skill
**テーマ**: Skill を活用して Azure テンプレート準拠の PowerPoint スライドを生成し、ADR の設計判断を反映する

## 前提条件

- [ラボ環境セットアップガイド](./setup/README.md)を完了していること
- `docs/adr/` に ADR が存在すること
- `infra/` にBicepファイルが存在すること

## シナリオ

ADR の設計判断や既存の成果物を入力に、Azure テンプレート準拠の PowerPoint スライドを生成する。構成図は可読性が低くなりがちなので、無理にスライドへ埋め込まない。

---

## ステップ1: ADR を踏まえたスライド生成

**Copilot CLIに入力:**

```
@docs/adr/ @infra/ @docs/design-infrastructure-diagram.drawio @docs/cost-estimate.xlsx の内容を踏まえて、Azure環境の概要スライドを docs/overview.pptx として作って。構成図は無理に画像挿入せず、別資料参照として扱ってよい。以下のページを必ず含めること:
1. 構成要素一覧（Bicepリソースベース）
2. 設計判断のポイント（ADRベース）
3. コスト概算（xlsxベース）
4. 参考資料（構成図や詳細資料への参照）
```

> [!TIP]
> **プロンプトの工夫**
>
> `@` で入力ソースを明示する。必須セクションを番号付きで指定する。構成図は情報源として渡すが、スライドには無理に貼り込まず、別資料参照に逃がせるようにしている。

> [!NOTE]
> 時間がかかる。

**期待する結果:**

- azure-pptx Skill が呼ばれ、ここまでの決定をまとめたスライドが生成される
- `docs/overview.pptx` として保存される
- Azure テンプレート準拠のデザインで、以下の内容が含まれる:
  - 構成要素一覧（Bicep で定義されたリソースから抽出）
  - 設計判断のポイント（ADR から抽出）
  - コスト概算（xlsx のデータを反映）
  - 参考資料スライド（`docs/design-infrastructure-diagram.drawio` などへの参照）

> [!IMPORTANT]
> - Skill が自動的に呼ばれたか確認する（ログに Skill 名が表示される）
> - 生成されたスライドを開いて、テンプレートのデザインが適用されているか確認する
> - 決定内容が適切に反映されているか確認する
> - 構成図を無理に縮小せず、別資料参照として整理されているか確認する

---

## まとめ

- **Skill の活用**: azure-pptx Skill で Azure テンプレート準拠のスライドを生成
- **ADR との連携**: `docs/adr/` の設計判断をスライドに反映
- **既存成果物の再利用**: 構成図（drawio）やコスト見積もり（xlsx）を `@` で入力に指定し、必要に応じて別資料参照として扱う
- **成果物は `docs/` へ**: 構成図（ラボ3）、コスト見積もり（ラボ4）、スライドと、すべての成果物が `docs/` に集約
- **Skill = 専門知識の定義**: Markdown ファイルでエージェントにテンプレートの使い方やレイアウトルールを定義
